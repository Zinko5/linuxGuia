Comandos para Docker
Iniciar docker sudo service docker start Ver las imagenes en el Sistema sudo docker images Ver contenedores activos sudo docker ps Ver contenedores activos e inactivos sudo docker ps -a Iniciar un contenedor sudo docker start nombreOIdContenedor Detener un contenedor sudo docker start nombreOIdContenedor Verificar si un contenedor está habilitado para iniciar automáticamente sudo docker inspect -f '{{.HostConfig.RestartPolicy.Name}}' nombreOIdContenedor *Si el resultado es always o "unless-stopped", el contenedor está habilitado para iniciar automáticamente, si es "no" o "on-failure", no está habilitado

Habilitar el inicio automático de un contenedor sudo docker update --restart always nombreOIdContenedor Deshabilitar el inicio automático de un servicio sudo docker update --restart no nombreOIdContenedor Descargar una imagen sudo docker pull nombreImagen:tag *Se pueden encontrar las imágenes y tags en dockerhub, si no se especifíca un tag (versión), se usará latest por defecto

Crear un contenedor a partir de una imagen descargada sudo docker run -d --name nombreDelContenedor nombreImagen:tag Ejecutar un comando en un contenedor en ejecución sudo docker exec -it nombreDelContenedor comandoAEjecutar
