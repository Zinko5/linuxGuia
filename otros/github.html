Usar git con github
Configuraciones basicas
# Primero es necesario tener git instalado y una cuenta en github

# Crear configuraciones globales para git en la computadora

# Identificarse git config --global user.name "Tu username"
git config --global user.email "Tu email" # Seleccionar vscode como el editor git config --global core.editor "code --wait" # Configurar el salto de linea para Linux git config --global core.autocrlf input # Si queremos vincular el directorio con github con HTTPS (no recomendable), crear una clave aqui, copiarla y guardarla, ya que no se podra volver a ver

# Si queremos vincular el directorio con github con SSH, crear una llave con: ssh-keygen # Podemos ponerle un nombre a la llave y una contraseña o dejar el nombre predeterminado y la contraseña vacia solo presionando Enter

# Copiar la llave copiando el resultado de: (Ctrl + shift + C para copiar en consola) cat ~/.ssh/NombredelaClaveCreada.pub # Pegar aqui y poner un titulo que identifique el equipo que creo la llave, guardar

# Cuando hagamos referencia al enlace del repositorio en github, se puede hacer a traves de HTTPS (no recomendable): https://github.com/TuUsuariodeGithub/NombredelRepositorio.git # O a traves de SSH: git clone git@github.com:TuUsuariodeGithub/NombredelRepositorio.git
Usar repositorio creado en github
# En consola, ubicarnos en el directorio en que deseamos que esté el repositorio y clonarlo: git clone git@github.com:TuUsuariodeGithub/NombredelRepositorio.git #Predeterminadamente, al clonar un repositorio se vincula con HTTPS

# Para cambiar el tipo de vinculacion, de HTTPS a SSH git remote set-url origin git@github.com:TuUsuariodeGithub/NombredelRepositorio.git # Para cambiar el tipo de vinculacion, de SSH a HTTPS git remote set-url origin https://github.com/TuUsuariodeGithub/NombredelRepositorio.git # Si lo vinculamos con HTTPS, cada vez que queramos hacer un push nos pedira el nombre de usuario (de github) y la contraseña, que sera la llave que creamos antes

# Ahora podemos usar los comando git add, git commit - m "mensaje indicando los cambios en el proyecto", git push, git pull, git branch y git checkout en el directorio
Comandos de git
# Muestra el estado actual del repositorio, archivos modificados, nuevos o eliminados git status # Agrega un archivo específico al área de preparación (staging) git add # Agrega todos los archivos modificados al área de preparación git add . # Crea un nuevo commit con los cambios en el área de preparación git commit -m "mensaje indicando los cambios en el proyecto" # Sube los commits locales al repositorio remoto git push # Descarga y fusiona los cambios del repositorio remoto git pull # Muestra todas las ramas del repositorio git branch # Cambia a una rama existente git checkout nombreDeLaRama # Crea una nueva rama y cambia a ella git checkout -b nombreDeLaRama # Sube una rama específica al repositorio remoto git push origin nombre-branch # Tomar en cuenta que no se mostrara nada al ejecutar "git branch" hasta que no se haga el primer push, que es cuando se creara la primera rama
